# 1-1 工作紀錄：Flask 網格地圖產生器 (Grid Map Generator)

## 專案目標
開發一個基於 Flask 的網格地圖網頁應用程式，允許使用者動態生成 `n x n` (5 ≤ n ≤ 9) 的網格，並透過順序點擊來設置「起點」、「終點」及「障礙物」。

## 已完成功能與實作細節

### 1. 後端基礎建置 (Flask)
- 建立 `app.py` 作為 Flask 伺服器入口。
- 設定根路由 `/` 渲染 `index.html` 樣板。
- 確保 Flask 環境變數與匯入正常運作（解決了先前的 Flask import 錯誤）。

### 2. 前端介面設計 (HTML/CSS)
- **架構 (`templates/index.html`)**：
  - 設計直覺的控制面板（維度設定輸入框、網格重置按鈕）。
  - 加入狀態指示卡片，動態提示使用者目前的設定步驟與剩餘障礙物數量。
  - 將所有提示與按鈕文字從「產生網格」優化調整為「重置網格」。
- **現代化互動美學 (`static/style.css`)**：
  - 採用舒適的 CSS 變數進行色彩管理，背景使用點狀圖案 (radial-gradient)。
  - 網格單元格加入了 hover 放大效果、波浪進場動畫 (`fadeIn`) 與點擊標記動畫 (`popIn`)。
  - 不同狀態的顏色區分明確：綠色（起點）、紅色（終點）、帶有斜線紋理的灰色（障礙物）。
  - 網格內加入置中的編號文字設計，並調整深色背景（起點、終點、障礙物）下的文字為白色以維護對比度。
  - 對行動裝置與小螢幕進行了 Responsive 響應式佈局調整。

### 3. 使用者互動邏輯 (JavaScript)
- **狀態機架構 (`static/script.js`)**：
  - 實作了清晰的狀態機：`WAITING` -> `PLACE_START` -> `PLACE_END` -> `PLACE_OBSTACLES` -> `DONE`，引導使用者按步驟進行設定。
- **輸入驗證與網格生成**：
  - 確保輸入的維度 `n` 限制在 5 到 9 之間，超過範圍會自動修正。
  - 透過 DOM 操作與 CSS Grid 動態渲染 `n x n` 的網格。
  - 每個單元格在生成時，會自動依序（由左至右、由上到下）賦予 `1` 到 `n*n` 的編號。
- **點擊事件控制**：
  - 限制最多設置 1 個起點、1 個終點，以及最多 `n - 2` 個障礙物。
  - 即時更新狀態指示文字與剩餘障礙物數量。
  - 防止已標記的單元格被重複點擊。

## 結論
1-1 的基礎網格地圖開發已完成，具備視覺吸引力與流暢的網頁操作體驗。新增的按鈕文案（重置網格）與網格編號能更清楚地幫助辨識座標位置，為後續的深度強化學習演算法（如路徑規劃）打下了良好的動態地圖展示基礎。
